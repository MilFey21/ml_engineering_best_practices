    Локальная установка ClearML Server¶

    Описание¶

    Этот документ описывает процесс установки и настройки ClearML Server локально для работы с проектом предсказания оттока клиентов.

    Зачем нужен локальный запуск?¶

    Локальный запуск ClearML Server может быть полезен в следующих случаях:

    Когда использовать локальный сервер:¶

    1. Конфиденциальность данных
    2. Работа с чувствительными или конфиденциальными данными
    3. Требования соответствия нормативным актам (GDPR, HIPAA и т.д.)

    4. Данные не могут покидать корпоративную сеть

    5. Полный контроль над инфраструктурой

    6. Необходимость полной кастомизации конфигурации
    7. Интеграция с внутренними системами компании

    8. Управление ресурсами и масштабированием

    9. Офлайн-разработка

    10. Работа в изолированной сети без доступа к интернету
    11. Разработка в средах без внешнего доступа

    12. Тестирование и разработка без зависимости от внешних сервисов

    13. Обучение и экспериментирование

    14. Изучение внутренней архитектуры ClearML Server
    15. Тестирование новых функций и конфигураций

    16. Разработка кастомных интеграций

    17. Высокая нагрузка

    18. Большое количество экспериментов и данных
    19. Избежание ограничений облачного сервиса
    20. Оптимизация производительности под конкретные задачи

    Когда НЕ нужен локальный сервер:¶

    1. Локальная разработка и прототипирование
    2. Облачный сервер ClearML (https://app.clear.ml) проще в настройке
    3. Не требует управления инфраструктурой

    4. Достаточно для большинства проектов

    5. Небольшие проекты

    6. Облачный сервер предоставляет все необходимые функции
    7. Бесплатный план достаточен для большинства задач

    8. Экономия времени на настройку и поддержку

    9. Командная работа

    10. Облачный сервер обеспечивает легкий доступ для всех членов команды
    11. Не требует настройки VPN или доступа к внутренней сети
    12. Автоматические обновления и поддержка

    Рекомендация: Для большинства случаев локальной разработки и обучения рекомендуется использовать облачный сервер ClearML. Локальный сервер стоит использовать только при наличии конкретных требований к конфиденциальности, контролю или офлайн-работе.

    Требования¶

    • Docker и Docker Compose установлены и запущены
    • Минимум 4GB свободной оперативной памяти
    • Минимум 10GB свободного места на диске

    Быстрый старт¶

    Важно: ClearML Server требует значительных ресурсов (минимум 4GB RAM, 10GB дискового пространства). Перед установкой убедитесь, что у вас достаточно ресурсов и что локальный сервер действительно необходим для ваших задач.

    Рекомендация: Если вы только начинаете работать с ClearML или работаете над небольшим проектом, рассмотрите возможность использования облачного сервера ClearML (https://app.clear.ml) - это проще и быстрее.

    Windows¶

    # Запуск сервера
pixi run clearml-server-start

# Или напрямую
.\scripts\clearml_server_start.ps1

    Linux/Mac¶

    # Запуск сервера
pixi run clearml-server-start-unix

# Или напрямую
bash scripts/clearml_server_start.sh

    Альтернатива: Использование официального docker-compose файла ClearML:

    Официальный файл уже скачан в проект как docker-compose.clearml-official.yml.

    Важно для Windows: Официальный docker-compose использует пути /opt/clearml/..., которые нужно адаптировать для Windows или использовать WSL2.

    Вариант 1: Использование WSL2 (рекомендуется для Windows)

    1. Установите WSL2 и Docker Desktop с поддержкой WSL2
    2. В WSL2 создайте директории: 
      sudo mkdir -p /opt/clearml/{logs,config,data}
sudo chmod -R 777 /opt/clearml
    3. Запустите сервер: 
      docker compose -f docker-compose.clearml-official.yml up -d

    Вариант 2: Использование упрощенной конфигурации

    Используйте файл docker-compose.clearml.yml из проекта, который использует Docker volumes вместо абсолютных путей:

    docker compose -f docker-compose.clearml.yml up -d

    Вариант 3: Использование облачного сервера (самый простой)

    Для локальной разработки рекомендуется использовать облачный сервер ClearML: - Зарегистрируйтесь на https://app.clear.ml - Получите credentials в Settings -> Workspace - Настройте клиент с этими credentials

    Это не требует локальной установки и настройки сервера.

    Что происходит при запуске¶

    1. Запускается Docker контейнер с ClearML Server
    2. Сервер настраивает все необходимые компоненты:
    3. MongoDB для хранения метаданных
    4. Elasticsearch для поиска
    5. Redis для кэширования
    6. File Server для артефактов
    7. Сервер становится доступным по адресам:
    8. Web UI: http://localhost:8080
    9. API Server: http://localhost:8008

    Первоначальная настройка¶

    1. Откройте http://localhost:8080 в браузере
    2. Создайте аккаунт (первый пользователь автоматически становится администратором)
    3. Перейдите в Settings -> Workspace -> Create new credentials
    4. Скопируйте Access Key и Secret Key

    Настройка клиента¶

    После получения credentials настройте клиент:

    # Через интерактивную настройку
clearml-init

# Или через переменные окружения
export CLEARML_API_HOST="http://localhost:8008"
export CLEARML_API_ACCESS_KEY="your-access-key"
export CLEARML_API_SECRET_KEY="your-secret-key"

    Windows PowerShell: 

    $env:CLEARML_API_HOST="http://localhost:8008"
$env:CLEARML_API_ACCESS_KEY="your-access-key"
$env:CLEARML_API_SECRET_KEY="your-secret-key"

    Управление сервером¶

    Просмотр логов¶

    pixi run clearml-server-logs

    Остановка сервера¶

    docker compose -f docker-compose.clearml.yml down

    Перезапуск сервера¶

    # Остановить
docker compose -f docker-compose.clearml.yml down

# Запустить снова
pixi run clearml-server-start
# или для Linux/Mac:
pixi run clearml-server-start-unix

    Хранение данных¶

    Данные ClearML Server хранятся в Docker volumes: - clearml-data - метаданные и конфигурации - clearml-logs - логи сервера - clearml-cache - кэш данных

    Для просмотра volumes: 

    docker volume ls | grep clearml

    Для удаления всех данных (осторожно!): 

    docker compose -f docker-compose.clearml.yml down -v

    Решение проблем¶

    Сервер не запускается¶

    1. Проверьте, что Docker запущен: 

      docker info

    2. Проверьте логи: 

      docker compose -f docker-compose.clearml.yml logs

    3. Проверьте, не заняты ли порты 8080 и 8008: 

      # Windows
netstat -ano | findstr :8080

# Linux/Mac
lsof -i :8080

    Не могу подключиться к серверу¶

    1. Убедитесь, что сервер запущен: 

      docker ps | grep clearml

    2. Проверьте, что порты проброшены правильно: 

      docker port clearml-server

    3. Попробуйте открыть http://localhost:8080 напрямую в браузере

    Проблемы с аутентификацией¶

    1. Убедитесь, что используете правильный URL сервера (http://localhost:8008 для API)
    2. Проверьте, что Access Key и Secret Key скопированы правильно
    3. Убедитесь, что credentials созданы в правильном workspace

    Альтернатива: Облачный сервер¶

    Если не хотите запускать локальный сервер, можно использовать облачный ClearML Server:

    1. Зарегистрируйтесь на https://app.clear.ml
    2. Создайте credentials в Settings -> Workspace
    3. Настройте клиент с этими credentials

    Преимущества облачного сервера: - Не требует локальных ресурсов - Всегда доступен - Автоматические обновления

    Недостатки: - Требует интернет-соединение - Ограничения на бесплатном плане
